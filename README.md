# مروارید فارم (Farm Management System)

سامانه‌ی مدیریت انبار، مصرف و خرید برای مجموعه‌ی پرورشی «مروارید فارم».
این برنامه یک اپلیکیشن تک‌صفحه‌ای (SPA) با پشتیبانی کامل از زبان فارسی، چیدمان
راست‌به‌چپ (RTL)، تقویم جلالی و حالت تاریک است.

> نکته‌ی امنیتی مهم: پیش از استقرار در محیط واقعی، حتماً فایل
> [`SECURITY.md`](./SECURITY.md) را بخوانید. در پیکربندی فعلی، عملیات داده با کلید
> `service_role` انجام می‌شود که نباید در نسخه‌ی production در مرورگر قرار گیرد.

## پشته‌ی فناوری
- **React 19** + **TypeScript 5.9**
- **Vite 7** (خروجی تک‌فایلی با `vite-plugin-singlefile`)
- **Tailwind CSS 4** (از طریق `@tailwindcss/vite`)
- **Zustand** برای مدیریت state (احراز هویت و UI)
- **Supabase** برای احراز هویت و پایگاه‌داده‌ی Postgres
- **react-router-dom 7** برای مسیریابی
- **react-hook-form** + **zod** برای فرم‌ها و اعتبارسنجی
- **date-fns-jalali** برای تقویم جلالی
- **sonner** برای پیام‌ها (toast)، **lucide-react** برای آیکن‌ها
- **xlsx** برای خروجی اکسل، **framer-motion** برای انیمیشن

## ساختار پوشه‌ها
```
src/
├── components/   کامپوننت‌های مشترک، UI، چیدمان و فرم‌ها
├── hooks/        هوک‌های داده (انبار، خرید، کاربران، فرمول‌ها، …)
├── lib/          کلاینت‌های Supabase و ذخیره‌سازی آفلاین
├── pages/        صفحات برنامه (به‌صورت lazy بارگذاری می‌شوند)
├── router/       تعریف مسیرها و محافظت نقش‌محور
├── store/        استورهای Zustand (authStore، uiStore)
├── types/        تعاریف TypeScript و تایپ‌های دیتابیس
├── utils/        توابع کمکی (logger، تاریخ جلالی، اعداد فارسی، اکسل، …)
└── validations/  اسکیماهای zod
scripts/          اسکریپت‌های کمکی (بررسی نشانگرهای merge و …)
```

## راه‌اندازی
پیش‌نیاز: **Node.js نسخه‌ی ۲۲ یا بالاتر** و **npm**.

```bash
# نصب وابستگی‌ها
npm install

# کپی فایل نمونه‌ی محیط و وارد کردن مقادیر واقعی
cp .env.example .env

# اجرای محیط توسعه
npm run dev
```

## متغیرهای محیطی
همه‌ی متغیرها باید با پیشوند `VITE_` تعریف شوند تا در برنامه قابل دسترسی باشند.
جزئیات و هشدارهای امنیتی در [`.env.example`](./.env.example) آمده است.

| متغیر | توضیح |
|---|---|
| `VITE_SUPABASE_URL` | آدرس پروژه‌ی Supabase |
| `VITE_SUPABASE_ANON_KEY` | کلید عمومی anon |
| `VITE_SUPABASE_SERVICE_ROLE_KEY` | ⚠️ کلید مدیریتی — نباید در production در مرورگر باشد (به SECURITY.md مراجعه کنید) |
| `VITE_ADMIN_USERNAME` | نام کاربری ادمین اولیه |
| `VITE_ADMIN_PASSWORD` | رمز عبور ادمین اولیه |
| `VITE_APP_VERSION` | نسخه‌ی نمایش‌داده‌شده در رابط کاربری |

## اسکریپت‌ها
| دستور | کار |
|---|---|
| `npm run dev` | اجرای سرور توسعه |
| `npm run build` | ساخت نسخه‌ی production (خروجی تک‌فایلی در `dist/`) |
| `npm run preview` | پیش‌نمایش خروجی build |
| `npm run typecheck` | بررسی نوع‌ها با `tsc --noEmit` (بدون تولید فایل) |
| `npm run check:conflicts` | بررسی وجود نشانگرهای merge conflict |

## ساخت و استقرار
```bash
npm run build
```
خروجی به‌صورت یک فایل `dist/index.html` تولید می‌شود (JS و CSS داخل آن inline
شده‌اند) که استقرار آن روی هر میزبان استاتیک ساده است.

> پوشه‌ی `dist/` دیگر در گیت نگه‌داری نمی‌شود و در هر بار build بازتولید می‌شود.

## نقش‌ها و دسترسی‌ها
| نقش | مسیر پایه | دسترسی |
|---|---|---|
| `admin` | `/admin` | همه‌ی امکانات: کاربران، فارم‌ها، انبار، خرید، فرمول‌ها، گزارش‌ها، نهاده‌ها |
| `supervisor` | `/supervisor` | مصرف، انبار، خرید، فرمول‌ها، گزارش‌ها، نقطه‌ی سفارش |
| `operator` | `/operator` | مصرف، انبار، خرید، گزارش‌ها |

دسترسی نقش‌محور در `src/router/routes.tsx` با کامپوننت `ProtectedRoute` اعمال
می‌شود. توجه کنید که این یک کنترل UI است و کنترل امنیتی واقعی باید با RLS در
Supabase انجام شود (به `SECURITY.md` مراجعه کنید).
